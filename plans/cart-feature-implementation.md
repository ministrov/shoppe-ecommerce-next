# План реализации фич добавления и удаления товаров в корзину

## Текущее состояние
Страница корзины (`app/cart/page.tsx`) использует статические моковые данные (`cartItemMocks`). Компоненты `CartItem` и `CartForm` отображают верстку без функциональности. Требуется реализовать динамическое управление корзиной: добавление товаров с других страниц, удаление, изменение количества, расчёт суммы.

## Цели
1. Реализовать глобальное состояние корзины (Redux Toolkit).
2. Добавить возможность добавления товара в корзину со страниц каталога и товара.
3. Реализовать удаление товара из корзины.
4. Обновить компоненты для работы с реальными данными.
5. Обеспечить интеграцию с бэкендом (если требуется).
6. Протестировать и задокументировать изменения.

## Архитектура

### Состояние корзины (Redux slice)
**Локация**: `store/features/cart/cartSlice.ts`

**Интерфейсы**:
```typescript
interface CartItem {
  id: string;           // уникальный идентификатор элемента корзины
  productId: number;    // ID товара из Product
  title: string;
  image: string | StaticImageData;
  price: number;        // числовая цена для расчётов
  quantity: number;
}

interface CartState {
  items: CartItem[];
  total: number;        // общая сумма
  loading: boolean;
  error: string | null;
}
```

**Действия**:
- `addToCart(product: Product, quantity?: number)`
- `removeFromCart(id: string)`
- `updateQuantity(id: string, quantity: number)`
- `clearCart()`
- `calculateTotal()` (автоматически при изменении items)

### Хук `useCart`
Создать хук `hooks/useCart.ts` для удобного доступа к состоянию и действиям корзины (аналогично `useFavorites`).

## План реализации

### Этап 1: Создание Redux slice и хука
1. Создать файл `store/features/cart/cartSlice.ts` с начальным состоянием, редьюсерами и действиями.
2. Добавить селекторы для получения items, total, loading.
3. Создать хук `useCart` в `hooks/useCart.ts`.
4. Обновить `store.ts` для включения слайса корзины.

### Этап 2: Добавление товара в корзину
1. На странице каталога (`app/catalog/page.tsx`) и странице товара (`app/catalog/[id]/page.tsx`) добавить кнопку "В корзину".
2. При клике диспатчить `addToCart` с данными товара.
3. Обработать логику увеличения quantity, если товар уже в корзине.
4. Показать уведомление (toast) об успешном добавлении.

### Этап 3: Удаление товара из корзины
1. В компоненте `CartItem` добавить кнопку удаления (крестик).
2. Передать пропс `onRemove` из родительской страницы, который диспатчит `removeFromCart`.
3. Обновить UI после удаления (оптимистичное обновление).
4. Добавить подтверждение удаления (опционально).

### Этап 4: Изменение количества
1. В `CartItem` реализовать компонент счётчика (кнопки +/-).
2. При изменении вызывать `updateQuantity`.
3. Запретить quantity < 1 (при quantity = 0 удалить товар).

### Этап 5: Обновление страницы корзины
1. Заменить моковые данные `cartItemMocks` на данные из Redux (`useCart`).
2. Передать в `CartItem` пропсы `onRemove` и `onChangeQuantity`.
3. Рассчитывать общую сумму на основе данных корзины и отображать в блоке "Итог".
4. Обработать пустую корзину (показ сообщения "Корзина пуста").

### Этап 6: Persistence и синхронизация
**Локальное состояние с localStorage**:
1. Добавить middleware в Redux slice для автоматического сохранения корзины в localStorage при каждом изменении.
2. При инициализации приложения загружать корзину из localStorage (в `initialState`).
3. Обеспечить совместимость структуры данных (версионирование при необходимости).
4. Очищать корзину при logout (опционально).

**Мокирование API** (для будущей интеграции):
- Создать файл `api/cart.ts` с функциями-заглушками, которые имитируют задержку и возвращают успешный результат.
- Использовать флаг `USE_MOCK_DATA` (как в других API) для переключения между моками и реальным API.
- Подготовить структуру thunk'ов, но пока использовать синхронные действия.

**Интеграция с бэкендом (отложена)**:
- Когда бэкенд будет готов, можно будет добавить thunk'и и заменить моки на реальные запросы.
