## Общее

| Что               | Кратко                                                   |
| ----------------- | -------------------------------------------------------- |
| **Фреймворк**     | Next.js 15 (App Router)                                  |
| **React**         | React 19 (только функциональные компоненты и хуки)       |
| **Язык**          | TypeScript                                               |
| **Стили**         | CSS Modules (`*.module.css`)                             |
| **Состояние**     | Redux Toolkit (глобальное), `useState` (локальное)       |
| **Анимации**      | Framer Motion                                            |
| **Иконки**        | Lucide React                                             |
| **Утилиты классов** | classnames              |
| **Линтер**        | ESLint с конфигурацией Next.js                           |
| **Комментарии**   | комментарии и описания — **на русском**                  |

## 1. Архитектура проекта

1. Исходники находятся в `src/`? **Нет**, используется структура Next.js App Router без `src/`. Основные папки:
   - `app/` – страницы и layout'ы (App Router)
   - `components/` – переиспользуемые компоненты
   - `layouts/` – компоненты макета (Header, Footer)
   - `store/` – Redux store, слайсы, thunk'и
   - `api/` – функции для работы с API
   - `interfaces/` – TypeScript-интерфейсы
   - `hooks/` – кастомные хуки
   - `helpers/` – вспомогательные функции
   - `styles/` – глобальные стили и шрифты
   - `public/` – статические ресурсы
   - `mocks/` – мок-данные для разработки
   - `docs/` – документация проекта

2. Каждый **компонент** – в отдельной папке `ComponentName/` внутри `components/` (например, `components/button/`), содержащей:
   - `ComponentName.tsx` – основной компонент
   - `ComponentName.module.css` – стили
   - `ComponentName.interface.ts` / `ComponentName.props.ts` – типы
   - (опционально) `ComponentName.test.tsx` – тесты
   - использовать именнованый экспорт по примеру:  `export const Rating = ({ ... }) => { ... }`

## 2. React

* **Только** функциональные компоненты, классовые компоненты запрещены.
* PascalCase для имён файлов и компонентов (`ProductCard.tsx`).
* Props неизменяемы; стейт всегда обновляем через set‑функции.
* Внешние эффекты – в `useEffect` с корректным списком зависимостей.
* Кастомные хуки – в папке `hooks/`, имя начинается с `use` (`useAuth.tsx`).
* Для условного рендеринга используйте тернарный оператор или `&&`.

## 3. TypeScript

* Все файлы компонентов и утилит должны иметь расширение `.ts` или `.tsx`.
* Интерфейсы и типы выносить в отдельные файлы в `interfaces/` или локально в `*.interface.ts`.
* Избегать `any`. Использовать `unknown` или конкретные типы.
* Для пропсов компонентов создавать отдельный тип/интерфейс.
* Использовать `import type` для импорта только типов.

## 4. Стили (CSS Modules)

* Стили компонента – в файле `ComponentName.module.css` в той же папке.
* Имена классов – в стиле kebab-case (`product-card`).
* **Нейминг классов:** используйте **плоские имена** (flat naming), а не БЭМ. Каждый класс должен быть самодостаточным и описывать элемент, а не его принадлежность к блоку. Примеры:
  - ✅ `.card`, `.image`, `.price`, `.header`, `.footer`
  - ❌ `.product-card__image`, `.footer__top`
* Исключение: если компонент уже использует БЭМ (например, `Footer`), можно оставить существующие классы для сохранения консистентности внутри компонента, но новые компоненты должны следовать плоскому неймингу.
* Глобальные стили – в `app/globals.css` и `styles/`.
* Для комбинирования классов использовать `classnames` или `clsx`.
* **Не** использовать инлайн-стили (`style={{ ... }}`), кроме динамических значений (например, `width: ${width}px`).
* **Не** использовать препроцессоры (Sass/Less) без согласования.

## 5. Состояние (Redux Toolkit)

* Глобальное состояние – через Redux Toolkit.
* Слайсы хранить в `store/features/`.
* Thunk'и – в `store/*Thunk/`.
* Использовать `createSlice`, `createAsyncThunk`.
* Селекторы – с помощью `createSelector` для мемоизации.
* Не дублировать состояние: если данные уже в store, не хранить локально.

## 6. Next.js (App Router)

* Страницы – в `app/[route]/page.tsx`.
* Layout – `app/layout.tsx`.
* Динамические маршруты – `app/[param]/page.tsx`.
* Загрузка данных – через `fetch` в Server Components, либо в `useEffect` в Client Components.
* Метаданные – через `export const metadata` в layout/page.
* Обработка ошибок – `error.tsx`, `loading.tsx`, `not-found.tsx`.
* Middleware – `middleware.ts` для защиты маршрутов и логики.

## 7. Анимации (Framer Motion)

* Для анимаций использовать `framer-motion`.
* Компоненты `motion.div`, `motion.span` и т.д.
* Плавные переходы между страницами – через `PageTransitions`.

## 8. Именование и структура

* Папки и файлы – kebab-case (например, `product-card`).
* Компоненты – PascalCase (`ProductCard`).
* Хуки – camelCase с префиксом `use` (`useAuth`).
* Функции и переменные – camelCase.
* Константы – UPPER_SNAKE_CASE.
* Интерфейсы и типы – PascalCase с суффиксом `Interface` или `Props` (например, `ProductInterface`, `ButtonProps`).

## 9. Код-стайл и линтинг

* Используется ESLint с конфигурацией Next.js.
* Перед коммитом запускать `npm run lint`.
* Форматирование – Prettier (опционально).
* Максимальная длина строки – 100 символов.
* Использовать одинарные кавычки для строк.
* Завершающая запятая в объектах и массивах.

## Ни в коем случае не добавляй:

- Class компоненты
- Styled Components / Emotion (только CSS Modules)
- Другие стейт-менеджеры (MobX, Zustand) без согласования
- Tailwind CSS
- Инлайн-стили на элементах (кроме динамических)
- Глобальные CSS-классы для стилизации компонентов
- UI-библиотеки (MUI, Ant Design) без согласования
- Дополнительные зависимости без обсуждения
